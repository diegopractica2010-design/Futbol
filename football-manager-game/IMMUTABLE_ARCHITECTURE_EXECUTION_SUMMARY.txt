📊 USAGE PATTERNS

### Pattern 1: Simple State Update
const state2 = state1.with({ version: 2 });

### Pattern 2: Transactional Update
const tx = new StateTransaction(state);
tx.apply(action1);
tx.apply(action2);
const { gameState } = tx.commit();

### Pattern 3: Pipeline with Hooks
const pipeline = new TransitionPipeline(state);
pipeline.before(ctx => validate(ctx.currentState));
pipeline.after(ctx => emit("updated", ctx.newState));
const { gameState } = pipeline.transition(action, "description");

### Pattern 4: Snapshot & Restore
const store = new SnapshotStore();
store.save(state, "checkpoint");
// later...
const restored = store.load(snapshotId);

### Pattern 5: Deterministic Replay
const engine = new ReplayEngine(store);
engine.replay(snapshotId, actions); // produces same result every time

### Pattern 6: Week Simulation
const sim = new SimulationEngineV2({ eventBus, matchSimulator });
sim.initialize(gameState);
const result = sim.advanceWeek(gameState, { weekSeed });
const newState = result.gameState;

📈 PERFORMANCE CHARACTERISTICS

Operation              | Complexity | Notes
──────────────────────┼────────────┼──────────────────────────
gameState.with()      | O(n)       | n = state fields (~10)
freeze()              | O(1)       | One-time cost
snapshot()            | O(1)       | Returns reference
applyAction()         | O(n)       | Depends on action
commit()              | O(1)       | Just marks state
replay()              | O(m)       | m = action count
validate()            | O(k)       | k = clubs (~20)

Memory:
   - No deep cloning (copy-on-write)
   - Shared immutable objects
   - Freeze has no memory cost
   - Snapshots: reference-based
