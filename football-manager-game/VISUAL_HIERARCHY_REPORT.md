# Visual Hierarchy Report

## Findings

- Many screens used identical cards for primary, secondary, and archival information.
- Long secondary sections caused the first read to feel like a dashboard rather than a football management game.
- Club identity existed through badges/colors, but not enough panels gave the club moment visual priority.

## Applied Hierarchy Changes

- Added `football-priority` panels for core screen reads.
- Added shared disclosure sections to separate primary decisions from advanced context.
- Added compact summary/disclosure CSS primitives for future screen-safe grouping.
- Moved deep-dive systems into labeled sections with clear football wording.

## Screen Outcomes

- Dashboard: club state and next fixture remain first; history/rival movement are secondary.
- Squad: plan and roster are first; tactics, psychology, academy, and player profile are progressive.
- Matches: score/HUD/visualizer first; live orders, substitutions, and full relato secondary.
- Calendar: next/current weeks first; full schedule available on demand.
- Market: scouting board first; economy/history secondary.
- Standings: league table first; cups/rankings secondary.
- Finances: cash and budgets first; infrastructure/staff/events secondary.
- Career/News/Rivals/Settings: hero and current action first; archives and technical payloads secondary.
